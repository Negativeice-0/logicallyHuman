## 🔄 **SYSTEM SWAP CHEAT SHEET**

### **1. Database Schema Changes**
```sql
-- Safe way to add columns
ALTER TABLE users ADD COLUMN IF NOT EXISTS new_column VARCHAR(255);

-- Safe way to modify columns
ALTER TABLE users ALTER COLUMN existing_column TYPE VARCHAR(500);

-- Never drop columns in production
-- Instead, mark as deprecated
ALTER TABLE users ADD COLUMN old_column_deprecated BOOLEAN DEFAULT TRUE;
```

### **2. Adding New Features (Feature Flags)**
```php
// In config.php
define('FEATURE_NEW_ML_MODEL', false);

// In your code
if (FEATURE_NEW_ML_MODEL) {
    // New feature code
} else {
    // Old feature code
}
```

### **4. Swapping ML Models Safely**
```javascript
// ML Module Loader - Safe swapping
class MLModuleManager {
    constructor() {
        this.modules = new Map();
        this.activeModule = 'sentimentV1';
    }

    registerModule(name, moduleClass) {
        this.modules.set(name, moduleClass);
    }

    setActiveModule(name) {
        if (this.modules.has(name)) {
            this.activeModule = name;
            console.log(`Switched to ML module: ${name}`);
        }
    }

    getActiveModule() {
        return new (this.modules.get(this.activeModule))();
    }
}

// Usage
const mlManager = new MLModuleManager();

// Register old and new models
mlManager.registerModule('sentimentV1', SentimentAnalyzerV1);
mlManager.registerModule('sentimentV2', SentimentAnalyzerV2);
mlManager.registerModule('customTrained', CustomTrainedModel);

// Switch safely
mlManager.setActiveModule('customTrained');
const analyzer = mlManager.getActiveModule();
```

### **5. A/B Testing New Features**
```php
// In config.php
class FeatureToggle {
    private static $features = [
        'new_ml_model' => 50, // 50% of users get new model
        'redesign' => 25,     // 25% of users get redesign
    ];

    public static function isEnabled($feature, $userId) {
        if (!isset(self::$features[$feature])) return false;

        // Deterministic based on user ID
        $hash = crc32($userId . $feature);
        return ($hash % 100) < self::$features[$feature];
    }
}

// Usage
if (FeatureToggle::isEnabled('new_ml_model', $_SESSION['user_id'])) {
    // Show new ML model
} else {
    // Show old ML model
}
```

### **8. User Preference System**
```php
// Add user_settings table
CREATE TABLE user_settings (
    user_id INTEGER REFERENCES users(id),
    setting_key VARCHAR(100),
    setting_value TEXT,
    PRIMARY KEY (user_id, setting_key)
);

// Get user preference
function getUserSetting($userId, $key, $default = null) {
    $db = getDB();
    $stmt = $db->prepare("SELECT setting_value FROM user_settings WHERE user_id = :user_id AND setting_key = :key");
    $stmt->execute([':user_id' => $userId, ':key' => $key]);
    $result = $stmt->fetch();
    return $result ? $result['setting_value'] : $default;
}
```

## 🎯 **KEY TAKEAWAYS**

1. **Always use feature flags** for new functionality
2. **Never delete database columns** - mark as deprecated
3. **Use interface patterns** for swappable components (like ML models)
4. **Keep user data separate** from application logic
5. **Have rollback plans** for every deployment
6. **Test with real user IDs** before full rollout
7. **Monitor performance** after any swap
8. **Document all changes** for future reference

This approach lets you safely experiment, customize, and scale your application without breaking existing functionality! 🚀
