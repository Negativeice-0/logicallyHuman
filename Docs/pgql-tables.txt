The issue is that the **`public` schema doesn't exist** anymore since you deleted it. You need to **recreate the schema first** before creating tables in it.

### ✅ Solution: Create the Schema

Run these commands in order:

```sql
-- Create the public schema
CREATE SCHEMA public;

-- Set it as default
SET search_path TO public;

-- Now create your tables
CREATE TABLE IF NOT EXISTS users (
    id SERIAL PRIMARY KEY,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    role VARCHAR(20) DEFAULT 'user',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS posts (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    title VARCHAR(255) NOT NULL,
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE IF NOT EXISTS comments (
    id SERIAL PRIMARY KEY,
    user_id INTEGER REFERENCES users(id),
    post_id INTEGER REFERENCES posts(id),
    content TEXT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

-- Grant all privileges
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO logic_user;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO logic_user;
GRANT ALL PRIVILEGES ON ALL FUNCTIONS IN SCHEMA public TO logic_user;

-- Set default privileges for future tables
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL PRIVILEGES ON TABLES TO logic_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL PRIVILEGES ON SEQUENCES TO logic_user;

-- Make sure logic_user owns the tables
ALTER TABLE users OWNER TO logic_user;
ALTER TABLE posts OWNER TO logic_user;
ALTER TABLE comments OWNER TO logic_user;
