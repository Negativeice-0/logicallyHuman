You need to grant your PostgreSQL user (`logic_user`) permissions on all tables in your database. Here are the commands to fix this:

## Grant All Permissions to Your User

```bash
# Connect to PostgreSQL as the postgres superuser
sudo -u postgres psql

# Then run these commands in the PostgreSQL prompt:
```

```sql
-- Connect to your database
\c logical_db

-- Grant all privileges on the database
GRANT ALL PRIVILEGES ON DATABASE logical_db TO logic_user;

-- Grant all privileges on all tables in the public schema
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO logic_user;

-- Grant all privileges on all sequences (for auto-increment IDs)
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO logic_user;

-- Grant usage on the schema itself
GRANT USAGE ON SCHEMA public TO logic_user;

-- Set default privileges for future tables
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO logic_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON SEQUENCES TO logic_user;

-- Exit PostgreSQL
\q
```

## Quick One-Liner Script

Or run this as a bash script:

```bash
sudo -u postgres psql -d logical_db -c "
GRANT ALL PRIVILEGES ON DATABASE logical_db TO logic_user;
GRANT ALL PRIVILEGES ON ALL TABLES IN SCHEMA public TO logic_user;
GRANT ALL PRIVILEGES ON ALL SEQUENCES IN SCHEMA public TO logic_user;
GRANT USAGE ON SCHEMA public TO logic_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON TABLES TO logic_user;
ALTER DEFAULT PRIVILEGES IN SCHEMA public GRANT ALL ON SEQUENCES TO logic_user;
"
```

## Verify Permissions

Check if permissions were granted correctly:

```bash
sudo -u postgres psql -d logical_db -c "\dp"
```

This will show you the access privileges for all tables.

## If You Need to Grant Permissions on Specific Tables

If you only want to grant permissions on specific tables:

```sql
-- For the photos table specifically
GRANT ALL PRIVILEGES ON TABLE photos TO logic_user;

-- For the users table
GRANT ALL PRIVILEGES ON TABLE users TO logic_user;

-- For any sequences (like photo_id_seq)
GRANT ALL PRIVILEGES ON SEQUENCE photos_id_seq TO logic_user;
GRANT ALL PRIVILEGES ON SEQUENCE users_id_seq TO logic_user;
```

After running these commands, your admin panel should work properly without permission errors. The error occurred because the `logic_user` didn't have SELECT, INSERT, UPDATE, or DELETE privileges on the `photos` table.
