You're getting this error because the `logic_user` doesn't own the `users` table. You need to **change ownership** or connect as the owner (usually `postgres`).

---

### 🔧 Fix: Connect as PostgreSQL Superuser

```bash
psql -h localhost -U postgres -d logical_db
```

When prompted, enter the **postgres password** (the one you set during PostgreSQL installation).

Then run your SQL commands:

```sql
ALTER TABLE users ADD COLUMN role VARCHAR(20) DEFAULT 'user';
UPDATE users SET role = 'user' WHERE role IS NULL;
UPDATE users SET role = 'admin' WHERE id = 1;
```

---

### 🔐 Alternative: Change Table Ownership

If you want `logic_user` to own the table, connect as `postgres` and run:

```sql
ALTER TABLE users OWNER TO logic_user;
```

Then you can use `logic_user` to modify the table in the future.
